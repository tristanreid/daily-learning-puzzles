# Puzzle Feedback

Append notes here any time. The weekly generator **reads this whole file** before writing new
lessons and adjusts accordingly. Keep entries short; reference lesson numbers when relevant.

Format (newest at the top is fine):

```
## YYYY-MM-DD
- L<number> or general: <what to change and why>
```

Examples of useful feedback:
- "L5 was too easy — fast-forward the accumulator material."
- "Stop using Python for examples in Stage 3; Scala pattern matching is clearer there."
- "Re-run this week: the fold lessons assumed I remembered map. Make them standalone."
- "I want two lessons per concept in Stage 7 — go slower on work/span."

Standing preferences (edit freely):
- Tone: direct, concrete, a little playful. Respect that the reader is an experienced dev.
- Examples: Python / TypeScript / Scala-flavored pseudocode by default. Explain any other language.
- Length: 5–10 minutes per puzzle. Split rather than pad.
